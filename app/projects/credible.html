my role

Product designer

the challenge

efficiency is not the real problem

This is a redesigning of the web version of sign up form. Credible's current form follows most of the best practice of web form design (single column design, breaking down into 3 steps, labels on the top left, etc.) The current form is already pretty efficient, so in order to have more users finish the form, maybe efficiency is not the real problem.

understand users

people are afraid of forms

I took a step back and start to ask myself very basic questions like: Why people are not willing to finish the form? - Because I don't like forms. Why don't people like forms? - Because it reminds of tons of tedious paperwork. Below are the search results of word "form" on Google image. These forms are always very long, difficult to understand, requires you a lot of sensitive data. But most of our current web forms still use this kind of format (a rectangular field with labels on it), which will give users these bad feelings subconsciously.

solutions

What if we make the forms do not look like forms?

With the insight "People are afraid of form", I started to ask myself "What if we make the form not looks like a form?", so that users can overcome the mindset that they are filling the form and forget the subconscious uneasiness. So I came up with two concepts as below.

nature language input form

Natural language input form converts traditional web form into several uncompleted sentences. A whole sentence provide a humanity feel, comparing with rectangle input fields and labels.

conversational UI

Using conversational UI creates a better emotional connection with users, which makes users feel like we are here to help them. On the other hand, navigating through different fields is painful on mobile and conversational form can solve this problem too. Though, it may not be an efficient way to input data.

next steps

how to evaluate these assumptions

We are still in the exploratory phase of this project, and we don't have the answer if these solutions will perform better or not. But we are planning to run A/B testing to validate our hypothesis.

internship highlights

Being in a really fast-paced start-up company, I got the chance to work on different types of projects. Below are some of the highlighted projects I've worked on during my internship.

Credible Mobile Navigation Redesign

The initial project was to add a new function on Credible's mobile offer dashboard, but the original design is not scalable. On page level, there are two full-width buttons as the entry point of two features. The original header already has two menus but some links within in the menu are not helpful for users, so we decide to rethink the whole mobile header and navigation system to help users navigate through the sites and use our new features easily. I went through several iterations and these are some key considerations:

  • Remove links that are not used frequently, based on data from Google Analytics
  • Group links to reduce users' cognitive load and reorder the links according to data
  • Improve the scalability to make sure adding new features in the further won't break user's experience
  • Explored and prototyped different transition and interaction of menu.

Personal Loan Email Template

The old Credible personal loan email template is pretty outdated. The visual style and layout is not consistent with other product's email template. I redesigned and developed Credible's personal email template to make sure it speaking to Credible's visual language and provide valuable information to users.

Improve designer - engineer Collaboration Process

Last but not least, when I joined Credible, the design team used PhotoShop to deliver designs to engineers for inspection. But none of the designers are using PhotoShop as their primary design tool. We all have to convert sketch file to AI files and then save as PSD files. And sometimes PSD files could be very big and often crash. I introduced a spec tool - Zeplin to both design and engineer team and it improves the collaboration very well. Both design and engineer team are very happy to use it.
